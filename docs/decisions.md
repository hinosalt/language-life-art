# Decisions

## 2026-02-18: 生成方式

- 仮説:
  - 言語表現の無限性は、固定文ではなく「語彙 + テンプレート + 文片化」による再構成で担保できる。
- 根拠:
  - 静的サイトでも実現可能で、GitHub Pagesとの整合が高い。
  - 語彙多層化とテンプレート群の交差で、入力UIなしでも毎回異なる軌道を作れる。
- 代替案:
  - LLM API接続による生成
  - 音声入力を起点にした生成
- 結論:
  - 初版は完全クライアントサイド生成とし、無限ループ/低依存/公開容易性を優先。

## 2026-02-18: 公開方式

- 仮説:
  - GitHub Pages + Actions の標準ワークフローが最短で安定運用できる。
- 根拠:
  - レポジトリ運用と公開が一体化できる。
- 結論:
  - `.github/workflows/pages.yml` を追加し、`main` pushで自動デプロイ。

## 2026-02-18: 体験インタラクションの制約

- 仮説:
  - 体験の純度を上げるため、入力UIを廃止し、観察とポインタ移動のみで成立させるべき。
- 根拠:
  - 「生命活動は自然現象」というコンセプトは、能動入力よりも連続観測と微小干渉の方が一致する。
- 代替案:
  - テキスト入力で語彙を手動追加
  - ボタンで再生/停止や文片発火
- 結論:
  - AI APIなし・入力UIなし。ポインタ軌跡のみを干渉パラメータとして扱う。

## 2026-02-18: 文言多様性と関係線ロジック強化

- 仮説:
  - 単一言語・単一線種では抽象体験が早期に飽和するため、多言語文法と複数線種の関係可視化が必要。
- 根拠:
  - 多言語語彙 + 文型モード（命題/連鎖/方程式/コーラス/誓約）で組合せ数を拡張できる。
  - ノード間関係スコアで `直線 / グラフ線 / 波線` を切替えると、意味関係が視覚的に層化される。
- 代替案:
  - 線種を固定して色だけ変える
  - 言語を日本語のみに固定する
- 結論:
  - 多言語生成・記憶再結合・3線種可視化を採用し、没入と無限性を優先する。
